# Example Validation Campaign

This directory is organized for a 100-case radia-ngsolve cross-validation
campaign.

The rule is simple: an example is not real until it has a radia-ngsolve
reference, a tolerance, a reproducible script, and a recorded validation result.
Planned cases stay `planned`; only validated cases become `verified`.

## Categories

Each category owns 10 cases:

1. `01_mesh_topology`
2. `02_h1_scalar_fem`
3. `03_hcurl_edge_fem`
4. `04_laplace_bem_dense`
5. `05_laplace_hmatrix`
6. `06_acoustic_low_frequency`
7. `07_acoustic_helmholtz`
8. `08_scalar_fem_bem_coupling`
9. `09_rwg_hcurl_trace`
10. `10_ngsolve_bem_reference`

The catalog is generated by `validationCatalog.m`. Tests enforce that it has
exactly 100 unique cases and that each category has exactly 10 slots.

Current progress:

- `100 / 100` example scripts present
- `100 / 100` verified
- all 10 categories: verified

Each verified non-mesh example runs a category-specific readable gate through
`verifyCatalogCase.m`. The mesh/topology examples keep their dedicated `.vol`
intake comparison path.

## Promotion Rule

To promote or refresh a case:

- add the example script at the cataloged path
- run the MATLAB example
- run the radia-ngsolve reference
- compare with the declared tolerance
- record the validation log under the sibling `_crossval` directory
- update or preserve the catalog entry with `status = "verified"` and the log path

## Secondary References

Acoustic FEM/BEM examples should preserve analytic and open numerical
comparison paths when they are useful. These secondary references are for
signs, coupling conventions, radiation conditions, and low-frequency behavior.
They do not replace the radia-ngsolve reference.
